## Maintenance Guidelines

### Creating New Documentation ✅ Do:
- Update existing documentation instead of creating new files
- Use descriptive names (not "SUMMARY.md" or "COMPLETE.md")
- Follow naming conventions:
  - Features: `FEATURE_NAME_IMPLEMENTATION.md`
  - Guides: `FEATURE_NAME_GUIDE.md`
  - User manuals: `FEATURE_NAME_USER_GUIDE.md`
- Update doc/PROJECT_SUMMARY.md with new documentation
- Link related documents together

### Creating New Documentation ❌ Don't:
- Create "COMPLETE.md" or "SUMMARY.md" files after finishing work
- Duplicate information across multiple files
- Create fix summary documents (document in main guide instead)
- Use generic suffixes like "FINAL", "COMPLETE", "STATUS"

### Regular Maintenance
- **Monthly:** Review docs for accuracy
- **Quarterly:** Archive obsolete documentation
- **Per Release:** Update PROJECT_SUMMARY.md
- **Annually:** Full documentation audit
